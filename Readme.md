# Git Commands for DevOps Engineers

This README file provides a list of essential Git commands that are commonly used by DevOps engineers.

## Basic Commands

1. **git clone <repository_url>** - Clone a repository to your local machine.
2. **git add <file_name>** - Add a file to the staging area.
3. **git commit -m "commit message"** - Commit changes with a message.
4. **git push origin <branch_name>** - Push changes to the remote repository.
5. **git pull origin <branch_name>** - Pull changes from the remote repository.
6. **git status** - Check the status of your working directory and staging area.
7. **git log** - View the commit history.
8. **git branch** - List all branches in the repository.
9. **git checkout <branch_name>** - Switch to a different branch.
10. **git merge <branch_name>** - Merge changes from one branch into another.
11. **git remote -v** - List all remote repositories.
12. **git fetch** - Download objects and refs from another repository.
13. **git reset --hard HEAD** - Discard all local changes   and reset to the last commit.
14. **git stash** - Temporarily save changes that are not ready to be committed.
15. **git stash pop** - Apply the most recently stashed changes and remove them from the stash list.
16. **git tag <tag_name>** - Create a new tag.
17. **git push origin <tag_name>** - Push a tag to the remote repository.
